# 用語集

| 用語 | 意味 |
| ---- | ---- |
| フロントエンド | ブラウザに表示される画面部分。ユーザーが目で見て操作する部分 |
| バックエンド | 画面の裏側で動く処理。データの保存・取得などを担当 |
| データベース（DB） | データを永続的に保存しておく場所。ページをリロードしてもデータが消えないのはDBに保存されているから |
| API | 外部サービスの機能を使うための窓口。「リクエストを送ると結果が返ってくる」仕組み |
| APIキー | APIを使うための入場チケット。他人に見せてはいけない。`.env.local` に書いて管理する |
| 環境変数 | APIキーなどの秘密情報を安全に管理する仕組み。`.env.local` に書く |
| Supabase | データベース・認証・APIをまとめて提供する無料サービス。このカリキュラムで使うDBサービス |
| 認証（Auth） | 「あなたは誰か」を確認する仕組み。メール/パスワードでのログインなど |
| RLS | Row Level Security（行レベルのセキュリティ）。「自分のデータだけ操作できる」ルール。Supabaseで必須の設定 |
| middleware.ts | アプリの「玄関の見張り役」ファイル。全ページへのアクセス時にログイン状態を自動チェックする |
| Git | コードの変更履歴を管理するツール。「いつ・何を変えたか」を記録でき、過去に戻せる |
| GitHub | Gitで管理したコードをインターネット上に保存・共有できるサービス（コードのクラウドストレージ） |
| .gitignore | GitHubにアップしてはいけないファイルを指定するリスト。`.env.local` を必ず含める |
| Vercel | GitHubと連携してWebアプリを自動公開するサービス。pushするだけでURLが更新される |
| デプロイ | アプリをインターネット上に公開すること |
| コンポーネント | 画面の部品（ボタン・カード・フォームなど）のひとまとまり |
| ルーティング | URLに応じて表示する画面を切り替える仕組み |
| リポジトリ | コードをGitHubで管理するプロジェクトの保管場所 |
| アーキテクチャ | システム全体の設計図・構造 |
| Next.js | Reactをベースにしたフレームワーク。このカリキュラムで使うフロントエンドの基盤 |
| Tailwind CSS | クラス名でスタイルを当てるCSSフレームワーク。`className="text-red-500"` のように書く |
| TypeScript | JavaScriptに型チェックを追加した言語。`.ts` / `.tsx` ファイルで使う |
| localhost:3000 | 自分のパソコン上でアプリを動かして確認するURL。インターネットには公開されていない |
| push | ローカルのコードをGitHubにアップロードする操作 |
| MECE | Mutually Exclusive, Collectively Exhaustive。「漏れなく・重なりなく」。画面の洗い出しなどで使う考え方 |
| MVP | Minimum Viable Product。「最小限の機能で動くもの」。2日間で確実に作れる範囲に絞ったバージョン |
